# Campus Utilities SQL Database System

A graduate-level enterprise database project developed for IST 634 Enterprise Databases at Cleveland State University.

## Overview
This project is a normalized relational database system designed to monitor and analyze campus utility operations, including buildings, meters, suppliers, employees, readings, maintenance logs, and alerts.

## Features
- 3NF normalized schema
- Multi-entity relational design
- Complex SQL queries for reporting and analysis
- Stored procedures for automation
- Triggers for data integrity and business rules
- Indexing for performance optimization

## Tech Stack
- SQL
- MySQL
- ER Modeling
- Database Normalization
- Query Optimization

## Use Cases
- Utility consumption tracking
- Alert generation for abnormal readings
- Maintenance logging
- Supplier performance analysis
- Campus operations reporting

## Files
- `er_diagram.png` – entity relationship diagram
- `schema.sql` – table definitions and sample data
- `queries.sql` – complex business queries
- `procedures.sql` – stored procedures and triggers

## Academic Context
Completed as a final project for IST 634 Enterprise Databases.
